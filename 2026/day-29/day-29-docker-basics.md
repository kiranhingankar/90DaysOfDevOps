# What is Docker?

Docker is a containerization platform that allows developers to build, package, and run applications in isolated environments called containers.

Docker ensures applications work consistently across:
- Development
- Testing
- Production

---

# What is a Container?

A container is a lightweight package that contains:
- Application code
- Dependencies
- Libraries
- Runtime environment

Containers help eliminate the problem:

❌ “It works on my machine.”

---

# Why Do We Need Containers?

Before containers:
- Applications behaved differently on different systems
- Dependency conflicts were common
- Environment setup took a lot of time

Containers solve these issues by providing:
✅ Consistent environments  
✅ Faster deployments  
✅ Isolation between applications  
✅ Better scalability  

---

# Containers vs Virtual Machines

| Containers | Virtual Machines |
|---|---|
| Lightweight | Heavyweight |
| Share host OS kernel | Include full Guest OS |
| Faster startup | Slower startup |
| Low resource usage | High resource usage |
| Best for microservices | Best for complete OS isolation |

## Key Difference

- Containers virtualize the application layer
- VMs virtualize the hardware layer

# Docker Architecture

Docker architecture consists of:

## 1. Docker Client
The command-line interface where users run Docker commands.

Example:

```bash
docker run nginx
```

---

## 2. Docker Daemon
Background service responsible for:
- Building images
- Running containers
- Managing Docker objects

---

## 3. Docker Images
Read-only templates used to create containers.

Examples:
- Ubuntu Image
- Nginx Image

---

## 4. Docker Containers
Running instances of Docker images.

---

## 5. Docker Registry
Stores Docker images.

Example:
- Docker Hub

---

# Docker Architecture Flow

```text
User → Docker Client → Docker Daemon → Docker Hub
                                ↓
                          Containers
```

### Explanation
1. User executes Docker command
2. Docker Client sends request to Docker Daemon
3. Docker Daemon checks for image locally
4. If image is unavailable, it pulls from Docker Hub
5. Container gets created and started

# Running Real Containers

## Run Nginx Container

```bash
docker run -d -p 8080:80 nginx
```

### Usage
- `-d` → Runs container in detached/background mode
- `-p 8080:80` → Maps host port 8080 to container port 80

Access in browser:

```text
http://localhost:8080
```

---

## Run Ubuntu Container in Interactive Mode

```bash
docker run -it ubuntu
```

### Usage
- `-it` enables interactive terminal mode

Try commands inside container:

```bash
ls
pwd
apt update
```

Exit container:

```bash
exit
```

# Docker Commands Practiced

## List Running Containers

```bash
docker ps
```

### Usage
Displays currently running containers.

---

## List All Containers

```bash
docker ps -a
```

### Usage
Displays all containers including stopped ones.

---

## Stop a Container

```bash
docker stop <container_id>
```

### Usage
Stops a running container.

Example:

```bash
docker stop a1b2c3d4
```

---

## Remove a Container

```bash
docker rm <container_id>
```

### Usage
Deletes a stopped container.

---

## Run Container in Detached Mode

```bash
docker run -d nginx
```

### Usage
Runs container in background without blocking terminal.

---

## Give Custom Name to Container

```bash
docker run -d --name mynginx nginx
```

### Usage
Assigns custom name to container for easier management.

---

## Port Mapping

```bash
docker run -d -p 3000:80 nginx
```

### Usage
Maps:
- Host Port → 3000
- Container Port → 80

---

## Check Container Logs

```bash
docker logs mynginx
```

### Usage
Displays logs generated by container.

---

## Execute Command Inside Running Container

```bash
docker exec -it mynginx bash
```

### Usage
Opens bash terminal inside running container.

---

# Important Docker Commands Summary

| Command | Usage |
|---|---|
| `docker run` | Run a new container |
| `docker ps` | List running containers |
| `docker ps -a` | List all containers |
| `docker stop` | Stop container |
| `docker rm` | Remove container |
| `docker logs` | View container logs |
| `docker exec` | Execute command inside container |
